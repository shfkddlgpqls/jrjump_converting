# 📞 CRM Implementation Guide

## 1. 구현 전략
- **비동기 처리**: SMS/메일 발송은 응답 대기 시간이 길 수 있으므로 `@Async` 또는 메시지 큐(RabbitMQ 등) 도입을 고려하되, 1단계에서는 **DB INSERT 후 리턴**하는 방식으로 구현합니다. (레거시와 동일하게)
- **이력 관리**: 모든 발송 행위는 `CRM_LOG` 또는 별도 로그 테이블에 기록해야 합니다.

## 2. 레거시 호환
- **팝업 통신**: 기존 PHP 팝업이 `opener.reload()` 등을 호출한다면, React/Vue 프론트엔드에서도 동일한 동작(이벤트 버스 등)을 지원해야 합니다.
